# 🟢 Simple Questions

### 1. List all products with their category names.

**Expected concepts:**

- JOIN

**Query:**

```sql
SELECT p.product_name, c.category_name
FROM products p
JOIN categories c ON p.category_id = c.category_id;
```

### 2. Show all customers and the number of orders they have placed.

**Expected concepts:**

- LEFT JOIN
- COUNT()
- GROUP BY

**Query:**

```sql
SELECT c.customer_id, c.customer_name, COUNT(o.order_id) AS order_count
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id
GROUP BY c.customer_id, c.customer_name;
```

### 3. Find products that cost more than $100.

- WHERE

```sql
SELECT *

```


- Multiple joins
- SUM()
- GROUP BY
- ORDER BY


# 🔴 Hard Questions

### 13. Find customers whose spending is above the average customer spending.

**Expected concepts:**

- Subqueries
- Aggregation

**Query:**

```sql
WITH CustomerSpending AS (
    SELECT c.customer_id, SUM(oi.quantity * oi.unit_price) AS total_spent
    FROM customers c
    JOIN orders o ON c.customer_id = o.customer_id
    JOIN order_items oi ON o.order_id = oi.order_id
    GROUP BY c.customer_id
),
AvgSpending AS (
    SELECT AVG(total_spent) AS avg_spent FROM CustomerSpending
)
SELECT cs.customer_id, cs.total_spent
FROM CustomerSpending cs, AvgSpending av
WHERE cs.total_spent > av.avg_spent;
```

### 14. Find pairs of customers who bought at least one common product.

**Expected concepts:**

- Self joins
- Distinct combinations

**Query:**

```sql
SELECT DISTINCT c1.customer_name AS Customer_A, c2.customer_name AS Customer_B, p.product_name AS Shared_Product
FROM customers c1
JOIN orders o1 ON c1.customer_id = o1.customer_id
JOIN order_items oi1 ON o1.order_id = oi1.order_id
JOIN products p ON oi1.product_id = p.product_id
JOIN order_items oi2 ON p.product_id = oi2.product_id
JOIN orders o2 ON oi2.order_id = o2.order_id
JOIN customers c2 ON o2.customer_id = c2.customer_id
WHERE c1.customer_id < c2.customer_id;
```

### 16. Find the longest streak of consecutive months in which each customer made at least one order.

**Expected concepts:**

- Window functions
- Gaps-and-islands problem

**Query:**

```sql
WITH CustomerMonths AS (
    SELECT DISTINCT customer_id, DATE_TRUNC('month', order_date) AS order_month
    FROM orders
),
GroupedMonths AS (
    SELECT customer_id, order_month,
           ROW_NUMBER() OVER(PARTITION BY customer_id ORDER BY order_month) AS rn
    FROM CustomerMonths
),
Streaks AS (
    SELECT customer_id, order_month - INTERVAL '1 month' * rn AS grp
    FROM GroupedMonths
)
SELECT customer_id, COUNT(*) AS longest_streak
FROM Streaks
GROUP BY customer_id, grp
ORDER BY longest_streak DESC;
```

### 17. Find products frequently bought together.

**Expected concepts:**

```sql
SELECT p1.product_name AS Product_A, p2.product_name AS Product_B, COUNT(*) AS Times_Bought_Together
```

### 18. Rank customers by spending within each year.

**Expected concepts:**

- Window functions
- RANK()

**Query:**

```sql
WITH YearlySpending AS (
    SELECT c.customer_id, c.customer_name, EXTRACT(YEAR FROM o.order_date) AS order_year, SUM(oi.quantity * oi.unit_price) AS total_spent
    FROM customers c
    JOIN orders o ON c.customer_id = o.customer_id
    JOIN order_items oi ON o.order_id = oi.order_id
    GROUP BY c.customer_id, c.customer_name, EXTRACT(YEAR FROM o.order_date)
)
SELECT customer_id, customer_name, order_year, total_spent,
       RANK() OVER(PARTITION BY order_year ORDER BY total_spent DESC) AS rank
FROM YearlySpending;
```

### 19. Detect suspicious orders whose value is more than three times the customer's average order value.

**Expected concepts:**

- CTEs
- Aggregation
- Statistical comparison

**Query:**

```sql
WITH OrderValues AS (
    SELECT o.order_id, o.customer_id, SUM(oi.quantity * oi.unit_price) AS order_value
    FROM orders o
    JOIN order_items oi ON o.order_id = oi.order_id
    GROUP BY o.order_id, o.customer_id
),
AvgOrderValues AS (
    SELECT customer_id, AVG(order_value) AS avg_order_value
    FROM OrderValues
    GROUP BY customer_id
)
SELECT ov.order_id, ov.customer_id, ov.order_value, aov.avg_order_value
FROM OrderValues ov
JOIN AvgOrderValues aov ON ov.customer_id = aov.customer_id
WHERE ov.order_value > 3 * aov.avg_order_value;
```

### 20. Find customers whose order amounts are continuously increasing over time.

**Expected concepts:**

- Window functions
- LAG()

**Query:**

```sql
WITH CustomerOrders AS (
    SELECT o.customer_id, o.order_date, SUM(oi.quantity * oi.unit_price) AS order_value
    FROM orders o
    JOIN order_items oi ON o.order_id = oi.order_id
    GROUP BY o.customer_id, o.order_date
),
OrderDifferences AS (
    SELECT customer_id, order_date, order_value,
           LAG(order_value) OVER(PARTITION BY customer_id ORDER BY order_date) AS prev_order_value
    FROM CustomerOrders
)
SELECT customer_id
FROM OrderDifferences
GROUP BY customer_id
HAVING MIN(order_value - COALESCE(prev_order_value, 0)) > 0 AND COUNT(prev_order_value) > 0;
```

# ⚫ Very Hard (AI-Level Reasoning)

### 21. Recommend products to customers based on purchasing patterns.

**Expected concepts:**

- Collaborative filtering
- Self joins
- Similarity scoring

**Query:**

```sql
WITH CustomerProducts AS (
    SELECT DISTINCT o.customer_id, oi.product_id
    FROM orders o
    JOIN order_items oi ON o.order_id = oi.order_id
),
SimilarCustomers AS (
    SELECT cp1.customer_id AS target_customer, cp2.customer_id AS similar_customer, COUNT(*) AS similarity_score
    FROM CustomerProducts cp1
    JOIN CustomerProducts cp2 ON cp1.product_id = cp2.product_id AND cp1.customer_id != cp2.customer_id
    GROUP BY cp1.customer_id, cp2.customer_id
)
SELECT sc.target_customer, cp.product_id AS recommended_product
FROM SimilarCustomers sc
JOIN CustomerProducts cp ON sc.similar_customer = cp.customer_id
LEFT JOIN CustomerProducts cp_target ON sc.target_customer = cp_target.customer_id AND cp.product_id = cp_target.product_id
WHERE cp_target.product_id IS NULL
ORDER BY sc.target_customer, sc.similarity_score DESC;
```

### 22. Find customer churn probability.

**Expected concepts:**

- Recency
- Frequency
- Monetary value (RFM analysis)

**Query:**

```sql
WITH RFM AS (
    SELECT c.customer_id,
           CURRENT_DATE - MAX(o.order_date) AS recency,
           COUNT(DISTINCT o.order_id) AS frequency,
           SUM(oi.quantity * oi.unit_price) AS monetary
    FROM customers c
    LEFT JOIN orders o ON c.customer_id = o.customer_id
    LEFT JOIN order_items oi ON o.order_id = oi.order_id
    GROUP BY c.customer_id
)
SELECT customer_id, recency, frequency, monetary,
       CASE
           WHEN recency > 180 THEN 'Churned'
           WHEN recency > 90 THEN 'At risk'
           ELSE 'Active'
       END AS churn_status
FROM RFM;
```

### 23. Build a leaderboard of customers whose spending growth rate is highest over time.

**Expected concepts:**

- Window functions
- Year-over-year calculations
- Percentage growth

**Query:**

```sql
WITH YearlySpending AS (
    SELECT o.customer_id, EXTRACT(YEAR FROM o.order_date) AS order_year, SUM(oi.quantity * oi.unit_price) AS total_spent
    FROM orders o
    JOIN order_items oi ON o.order_id = oi.order_id
    GROUP BY o.customer_id, EXTRACT(YEAR FROM o.order_date)
),
Growth AS (
    SELECT customer_id, order_year, total_spent,
           LAG(total_spent) OVER(PARTITION BY customer_id ORDER BY order_year) AS prev_year_spent
    FROM YearlySpending
)
SELECT customer_id, order_year,
       (total_spent - prev_year_spent) / prev_year_spent * 100 AS growth_rate
FROM Growth
WHERE prev_year_spent IS NOT NULL
ORDER BY growth_rate DESC;
```

### 24. Find hidden communities of customers.

**Expected concepts:**

- Recursive CTEs
- Graph traversal

**Query:**

```sql
WITH RECURSIVE SharedProducts AS (
    SELECT DISTINCT c1.customer_id AS cust1, c2.customer_id AS cust2
    FROM orders o1
    JOIN order_items oi1 ON o1.order_id = oi1.order_id
    JOIN order_items oi2 ON oi1.product_id = oi2.product_id
    JOIN orders o2 ON oi2.order_id = o2.order_id
    WHERE o1.customer_id < o2.customer_id
),
Community AS (
    SELECT cust1 AS base_cust, cust2 AS connected_cust
    FROM SharedProducts
    UNION
    SELECT c.base_cust, sp.cust2
    FROM Community c
    JOIN SharedProducts sp ON c.connected_cust = sp.cust1
)
SELECT base_cust AS community_id, array_agg(DISTINCT connected_cust) AS members
FROM Community
GROUP BY base_cust;
```

### 25. Determine the minimum number of products that generate 80% of total revenue (Pareto analysis).

**Expected concepts:**

- Window functions
- Running totals
- Percentage calculations

**Query:**

```sql
WITH ProductRevenue AS (
    SELECT p.product_id, SUM(oi.quantity * oi.unit_price) AS revenue
    FROM products p
    JOIN order_items oi ON p.product_id = oi.product_id
    GROUP BY p.product_id
),
TotalRev AS (
    SELECT SUM(revenue) AS total_revenue FROM ProductRevenue
),
RunningRevenue AS (
    SELECT product_id, revenue,
           SUM(revenue) OVER(ORDER BY revenue DESC) AS running_rev,
           (SELECT total_revenue FROM TotalRev) AS grand_total
    FROM ProductRevenue
)
SELECT COUNT(*) AS products_for_80_percent
FROM RunningRevenue
WHERE running_rev - revenue < 0.8 * grand_total;
```
